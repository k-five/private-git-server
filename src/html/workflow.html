workflow

  1. Our teams, developers code, commit new code and push to the private git server.

  2. Private git server receives new code and does a git merge for new code into old one.

  3. If merge succeeded, it will be pushed to gitlab (private) repository, or github (public) repository.

  4. After successful merge, if there is a test, it cab be run.

  5. After successful merge, if there is a build, it cab be run.

  6. Finally when there is no error, a change notification is delivered to production server to tell, to synchronize itself with last changes.
